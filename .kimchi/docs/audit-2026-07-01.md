## Catégorie 1 — Frontend request execution pipeline

### Stale closures & re-renders
- `use-request-store` n'utilise pas `useShallow` → callbacks recréés à chaque render → effets re-déclenchés
- `aiTabHandlers` (useMemo) dépend de `activeTab` (objet entier) → recréé à chaque keystroke
- `saveAllTabs`/`forceCloseTab`/`closeToRight` : `setActiveTabId` appelé dans le reducer `setTabs` → StrictMode le déclenche 2×
- `addNewTab` : collision possible sur nom (`tabs.length + 1` non monotone)
- `syncActiveTabToAiStore` : ne reset pas `setLastResponse(null)` quand on quitte un tab avec réponse

### Try/catch manquants → unhandled rejections
- `runCollectionBackground` (for-loop sans try/catch → UI bloquée si une requête throw)
- `sendSpecificRequest` (utilisé par 6 callers)
- `sendRequest`/`sendAndSave`/`sendAndDownload` (handlers UI)
- `loadAndSendRequest` (depuis CollectionsModal)
- Pending-collection effect IIFE
- `executeRequestWrapper` ne catche pas les throws de `runScript`

### Type holes
- `TauriFetchResponse.headers` typé `Record<string,string>` alors que Rust envoie `Vec<(String,String)>` → `Object.fromEntries` perd silencieusement les doublons (Set-Cookie, multi-value Vary, Link)
- `useRequestStore()` retourne un objet recréé à chaque render (merge+full rebuild)
- Plusieurs `as unknown as X` masquent des méthodes manquantes
- `parseJsonSafe` typé `Promise<any>`

### Native vs Web divergence
- Tabs persistés via `persistence` (IDB-only), pas via `storageAdapter` (Tauri FS → IDB) → en mode Tauri, les tabs ne vont jamais sur le disque
- `process.env.NODE_ENV` baked-in au build → header debug jamais envoyé en Tauri prod
- `exportActiveRequest` re-détecte Tauri inline au lieu d'utiliser `state.nativeMode`

### Storage fallback safety
- Tauri FS adapter retourne `null` seulement si message contient `"No such file"` → sur Windows `"os error 2"` ne match pas → throw propagé → fallback IDB → wipe mémoire
- Save/load asymétrique : si save primaire réussit, secondaire jamais écrit → divergence silencieuse
- `flushSave` après 3 retries jette `pendingStore` → données perdues si tab fermée entre-temps

## Catégorie 2 — Backend sécurité (Next.js + Tauri)

### SSRF — `/api/proxy/route.ts`
- **A1 [HIGH]** IPv6 : seul `::1` bloqué, manque `fe80::/10`, `fc00::/7`, `fd00::/8`, IPv4-mapped IPv6, CGNAT `100.64.0.0/10`, metadata cloud IPv6
- **A2 [HIGH]** String-prefix matching : faux positifs (`0.0.0.0.example.com`), faux négatifs (`2127.0.0.1`), première réponse DNS uniquement
- **A3 [MEDIUM]** DNS rebinding TOCTOU : resolve puis fetch → fenêtre où le DNS peut flipper
- **A4 [LOW]** Pas de check explicite pour `169.254.169.254` (AWS/GCP/Azure metadata)
- **A5 [LOW]** `ALLOW_LOCAL_HOSTS` désactive tout le SSRF guard

### Rate limiting
- **B1 [HIGH]** `X-Forwarded-For`/`X-Real-IP` acceptés comme clé → spoofable trivialement
- **B2 [HIGH]** In-process Map → chaque lambda Vercel a son propre compteur → cap effectif = 100 × N
- **B3 [MEDIUM]** Memory growth : pas de cap sur `entries.size`
- **B4 [MEDIUM]** Mock rate limiter unbounded
- **B5 [LOW]** Counter incrémenté même après limite hit

### Auth & session
- **C1 [MEDIUM]** Pas de CSRF token sur routes state-mutating (postman-auth, github-import, mock/config, postman-import)
- **C2 [MEDIUM]** `/api/postman-auth/status` retourne `connected: true` sans re-valider la clé
- **C3 [MEDIUM]** DELETE sur postman-auth ne révoque pas la clé upstream (acceptable mais à documenter)
- **C4 [LOW]** GitHub OAuth state cookie sans préfixe `__Host-`
- **C5 [LOW]** GitHub OAuth sans PKCE
- **C6 [LOW]** Tokens Bearer stockés en clair dans les cookies

### Mock catch-all routing
- **D1 [LOW]** `generateStaticParams` + `force-dynamic` contradictoires
- **D2 [MEDIUM]** 404 leak : liste routes, workspaceId, debug config → enumeration disclosure
- **D3 [LOW]** Pas de shared-secret gate sur `/api/mock/*` déployé
- **D4 [LOW]** `addMockLog` est un no-op silencieux (feature cassée)

### Tauri command surface
- **F2 [MEDIUM]** `open_external` ouvre n'importe quel scheme (`file://`, `ms-settings:`, custom URI)
- **F3 [MEDIUM]** `start_capture_proxy` est un relais SSRF localhost
- **F4 [MEDIUM]** `fetch_proxy` sans allow-list de schemes ni SSRF block
- **F5 [LOW]** `rand_id()` non crypto-secure → collisions probables
- **F6 [LOW]** `forward_request_sync` spawn un runtime Tokio par requête → DoS facile
- **F7 [LOW]** Race sur `start_capture_proxy` (OnceLock check racy)

### Tauri capability & storage paths
- **G1 [HIGH]** `$APPDATA/**` ≠ chemin Rust écrit (`data_dir().join("reqly")`) → "forbidden path" sur Windows → fallback IDB silencieux
- **G2 [MEDIUM]** Capability ne couvre pas tous les paths réellement utilisés
- **G3 [MEDIUM]** Pas de `fs:allow-mkdir` permission
- **G4 [LOW]** CSP sans `https:` → bloque tout sauf IPC bridge (OK pour desktop)
- **G5 [LOW]** Capability `[main]` appliquée à toutes fenêtres futures

### Input validation
- **H1 [MEDIUM]** `/api/proxy-ai` SSRF + auth-leak (voir aussi catégorie AI)
- **H2 [MEDIUM]** Provider check string-trimmed mais non enum-validated
- **H3 [MEDIUM]** `/api/github-import` anonymous → abuse rate-limit GitHub
- **H4 [LOW]** `/api/postman-export` body unbounded
- **H5 [LOW]** `collectionId` interpolé sans `encodeURIComponent`
- **H6 [LOW]** AI chat metadata stockée sans cap de taille
- **H7 [LOW]** `/api/test-runner/run` amplifie le rate-limit

### WebSocket
- **I1 [INFO]** Pas de route `/api/ws/*` côté Next.js
- **I2 [MEDIUM]** Tauri `ws_connect` sans allowlist → accès services localhost (Redis 6379, ClickHouse 8123, etc.)
- **I3 [LOW]** `ws_get_status` accepte n'importe quel connection_id

### Miscellaneous
- **J1 [LOW]** Content-Length client-supplied non vérifié
- **J2 [LOW]** Erreurs upstream propagées verbatim (leak hostnames internes)
- **J4 [LOW]** `delete_mock_route` exposé mais dead code

## Incohérences transverses

- **Mock store** : double-écriture Rust (`data_dir/reqly/mock-routes.json`) et JS (`persistence.getItem("reqly-mock-routes")`) sans sync → peuvent diverger indéfiniment
- **CSP** : `connect-src` autorise `ipc:`, `http://ipc.localhost`, `tauri:`, `https://tauri.localhost` mais pas `https:` → tout HTTP externe doit passer par le bridge Rust
- **Persistance asymétrique** : `use-request-store.ts` utilise `storageAdapter` (Tauri FS → IDB), le reste utilise `persistence` (IDB only) → en Tauri, la majorité des données est dans WebView IDB (risque de perte sur reinstall)
- **API key defaults divergents** : ai-engine `gpt-4o`, ai-config `gpt-3.5-turbo`, proxy `gpt-4o-mini`
- **Type-time contracts** : `TauriFetchResponse.headers` (Record) ≠ Rust `Vec<(String,String)>` → perte silencieuse de doublons
